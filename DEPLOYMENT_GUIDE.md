# HealthBridge AI - Cloud Deployment Guide

This guide provides the exact commands to deploy your consolidated microservices to **Google Cloud Run** and the frontend to **Vercel**.

## 1. Backend Deployment (Google Cloud Run)

Before running these, ensure you have the [Google Cloud SDK](https://cloud.google.com/sdk/docs/install) installed and are logged in (`gcloud auth login`).

### Step 1: Set Project & Enable APIs
```powershell
$PROJECT_ID = "your-gcp-project-id" # CHANGE THIS
gcloud config set project $PROJECT_ID
gcloud services enable run.googleapis.com containerregistry.googleapis.com
```

### Step 2: Deploy Services
Run these from the root directory of the project:

**A. Patient Management Service**
```powershell
gcloud run deploy patient-service --source ./patient_service --region us-central1 --allow-unauthenticated --env-vars-file .env
```

**B. Clinical Intelligence Service**
```powershell
gcloud run deploy clinical-service --source ./clinical_service --region us-central1 --allow-unauthenticated --env-vars-file .env
```

**C. HealthBridge AI Core**
```powershell
gcloud run deploy healthbridge-ai --source ./healthbridge_ai --region us-central1 --allow-unauthenticated --set-env-vars GOOGLE_API_KEY=your_gemini_key
```

> [!IMPORTANT]
> Note the URLs generated by these commands (e.g., `https://patient-service-xyz.a.run.app`). You will need them for the frontend.

## 2. Frontend Deployment (Vercel)

The easiest way to deploy the frontend is using the Vercel Dashboard:

1. Go to [Vercel](https://vercel.com/new).
2. Import your GitHub repository: `Vishalkrishna3434/HealthCareBridgeAI`.
3. Set the **Root Directory** to `frontend`.
4. Add **Environment Variables**:
   - `VITE_PATIENT_SERVICE_URL` = [Your Cloud Run URL]
   - `VITE_CLINICAL_SERVICE_URL` = [Your Cloud Run URL]
   - `VITE_AI_SERVICE_URL` = [Your Cloud Run URL]
5. Click **Deploy**.

## 3. Post-Deployment Verification
Once deployed, your Vercel link (e.g., `https://healthcarebridge-ai.vercel.app`) will be your live production link.

### Troubleshooting Live Services
- Check logs in the GCP Console: `Cloud Run > [Service] > Logs`.
- Ensure CORS is handled (already updated to `allow_origins=["*"]` in your code).
